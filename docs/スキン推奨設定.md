# スキン推奨設定一覧 v1.0

スキンは単体では完成せず、**カスタマイザー設定との組み合わせ**で本来の姿になる。
納品・デモ・検証のたびに記憶に頼らないよう、スキンごとの推奨設定をここに集約する。

**運用ルール**
- スキンを新規制作したら、完成時にこの表へ1行追加する（スキン量産ワークフローの完了レポートに含める）
- 設定に「必須」があるスキンは太字で明記する（例：yukiの雪はモーション rich が必須）
- `—` は「どれでも成立する」の意味

| スキン | フォントペア | モーション | 見出しの装飾 | 子猫一覧 | ヒーロー | 備考 |
|---|---|---|---|---|---|---|
| default | —（非対応） | soft | line | grid | split | ベースの素の姿。線と影が両方付く問題は要検討（→宿題） |
| yuki | —（非対応） | **rich（必須）** | 要確認 | grid | 要確認 | **CTAの降雪はrich時のみ**。全面の粉雪ドットは透明度0.06の極薄仕様 |
| nostalgia | —（非対応） | 要確認 | 要確認 | grid | 要確認 | 写真の角丸は上2角のみ（2026-07-13修正） |
| standard-01 | classic / modern / soft | soft | line | grid | split / Photo | ペアが線/影の世界を持つ（classic=線、modern/soft=影）。3色は設定ブロックで納品時に指定 |

## 「要確認」の埋め方

該当スキンをカスタマイザーで表示し、制作時のスクショ（docs/briefs/refs/）と見比べて確定値を記入する。
確定したらこのファイルのバージョンを上げる。

## 宿題（ベーステーマ側・別セッションで扱う）

1. **defaultスキンの線+影の同時適用** — `_tokens.scss` が `--shadow-card`（実影）と `--border-card`（1px線）を両方既定で持つため、defaultカードに両方付く。どちらの世界に寄せるか決める。
2. **yuki：ヒーローの額縁ズレがc-reveal完了まで正しくない** — 出現アニメーションのtransformと額縁オフセットのtransformが衝突している疑い。表示直後はズレ量が不正で、出現完了時に正位置になる。yukiを次に触るセッションで修正
3. **推奨設定の仕組み化（構想）** — スキン登録配列に `recommended_mods` を持たせ、カスタマイザーでスキン切替時に推奨値との差分を管理画面に通知する。手動運用で不便が出てから実装を判断する。
